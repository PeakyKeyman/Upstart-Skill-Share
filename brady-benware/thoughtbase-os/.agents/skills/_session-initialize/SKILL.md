# Session Initialization Skill

## Core Directive

Do not front-load the full project brain. Start with the thin root OS layer, then retrieve only the relevant domain guidance.

## Required Startup Reads

Always load:

1. `docs/os/00-constitution.md`
2. `docs/os/10-router.md`

If the task is already classifiable, then load only the relevant domain handbook and any clearly necessary operator or product file.

## Documentation Maintenance Protocol

Thoughtbase uses current-state memory. If the task changes a durable opinion, update the narrowest canonical file that owns it.

## Routing Rule

Use `docs/os/10-router.md` to determine what to load next and `docs/os/policies/process/documentation-routing.md` when deciding where new durable facts belong.
