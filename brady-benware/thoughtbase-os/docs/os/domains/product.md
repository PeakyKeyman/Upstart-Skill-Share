# Product Domain Handbook

## Purpose

This handbook orients work that affects user-visible behavior, feature contracts, and durable workflow expectations.

## Current Stance

- Product truth belongs in `docs/prd.md`.
- Product docs should describe what the app promises to the user, not internal implementation preferences.
- If an implementation detail does not materially change the product contract, keep it out of product docs.

## Use This Handbook When

- capture, search, or thought-management behavior changes
- a new feature alters user-facing flows
- data relationships matter at the product-contract level

- `docs/prd.md`
