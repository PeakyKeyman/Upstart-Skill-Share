# Operations Domain Handbook

## Purpose

This handbook orients work that affects deployment, verification, repository execution workflow, and runtime operations.

## Current Stance

- Operations guidance should explain how to run and verify the system, not restate implementation worldview.
- Deployment policy belongs in an operations leaf policy.
- Verification is part of execution, not an optional afterthought.

## Use This Handbook When

- deploying or restoring the app
- defining operator workflows
- choosing how changes must be tested or verified
